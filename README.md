## What is Slyme OS?

Slyme OS is a custom Arch Linux distribution where **AI is a system primitive, not an app**.

Most "AI-powered" tools bolt a chatbot onto an existing system. Slyme OS is architected differently ‚ÄĒ a unified AI daemon (`slyme-ai`) runs as a systemd service at boot, exposes a Unix socket, and every component of the OS talks to it. The shell, the window manager, the file system, the editor, the clipboard ‚ÄĒ all AI-aware, all local, all private.


**No cloud. No API keys. No accounts. No data leaves your machine.**

### The AI Stack

**`slyme-ai` ‚ÄĒ The AI Daemon**

The architectural core. A Python systemd service running at boot, exposing `/run/slyme-ai.sock`. Every OS component gets AI via this single socket ‚ÄĒ swap models, change routing, add features without touching dependent tools.

- **Dual model routing** ‚ÄĒ `phi3:mini` (~80ms) for shell, `mistral:7b` for deep queries
- **System context injection** ‚ÄĒ every AI call auto-enriched with cwd, clipboard, git branch, recent commands, focused window
- **Degraded mode** ‚ÄĒ if Ollama is unavailable, daemon stays alive and returns structured errors. Nothing crashes.
- **Thread-safe** ‚ÄĒ 16 concurrent clients supported

## ūüĒß Installation

### Option A ‚ÄĒ Install to disk (recommended)

1. **Flash the ISO to USB**
```bash
# Replace /dev/sdX with your USB device ‚ÄĒ check with lsblk first
sudo dd if=slyme-os-0.9.9-x86_64.iso of=/dev/sdX bs=4M status=progress oflag=sync
```

2. **Boot from USB**
   - Select "Slyme OS 0.9.9 ‚ÄĒ Boot Live Session" from the boot menu

3. **Live session boots automatically**
   - Autologin as `slyme` (password: `slyme`)
   - SLIWM launches automatically
   - Open a terminal and run:
```bash
install-slyme
```

4. **Calamares installer guides you through**
   - Choose language, keyboard, timezone
   - Partition your disk (automatic or manual)
   - Create your user account
   - Install ‚ÄĒ fully offline, no internet required

5. **First boot**
   - All services start automatically
   - If internet is available, `phi3:mini` downloads in the background (~2GB)
   - Run `slyme-status` to check everything is running

### Option B ‚ÄĒ Test in QEMU before installing

```bash
# Create a virtual disk
qemu-img create -f qcow2 slyme-test.qcow2 30G

# Boot (BIOS mode)
sudo qemu-system-x86_64 \
  -m 4096 -smp 4 \
  -cdrom slyme-os-0.9.9-x86_64.iso \
  -hda slyme-test.qcow2 \
  -boot d -vga virtio -display gtk \
  -enable-kvm -cpu host \
  -net nic -net user

# Boot (UEFI mode)
cp /usr/share/edk2/x64/OVMF_VARS.4m.fd /tmp/slyme-uefi-vars.fd
sudo qemu-system-x86_64 \
  -m 4096 -smp 4 \
  -cdrom slyme-os-0.9.9-x86_64.iso \
  -hda slyme-test.qcow2 \
  -boot d -vga virtio -display gtk \
  -enable-kvm -cpu host \
  -net nic -net user \
  -drive if=pflash,format=raw,readonly=on,file=/usr/share/edk2/x64/OVMF_CODE.4m.fd \
  -drive if=pflash,format=raw,file=/tmp/slyme-uefi-vars.fd
```

### The `slime` Socket Protocol

Any script or tool can talk to the AI daemon:

```bash
# From shell
echo '{"action":"query","prompt":"explain git rebase"}' | \
  socat - UNIX-CONNECT:/run/slyme-ai.sock

# From Python
import socket, json
sock = socket.socket(socket.AF_UNIX, socket.SOCK_STREAM)
sock.connect("/run/slyme-ai.sock")
sock.sendall(json.dumps({"action":"shell","input":"find large files"}).encode() + b"\n")
response = json.loads(sock.recv(4096))
```

Available actions: `ping` `query` `shell` `explain_error` `status`

## ūüĒź Security

- All AI inference is local ‚ÄĒ prompts never leave your machine
- No telemetry, analytics, or crash reporting of any kind
- Ollama API bound to `127.0.0.1` only ‚ÄĒ not exposed on network
- Unix socket at `/run/slyme-ai.sock` with `chmod 666` ‚ÄĒ local access only
- Live session uses passwordless sudo (expected for live ISO, removed on install)
- Verify ISO integrity with SHA256 before flashing (hash on release page)

Found a security issue? Email [slyme@tutamail.com](mailto:slyme@tutamail.com) privately before opening a public issue.
